You are in **explore** mode for SSWA — the equivalent of `/opsx:explore`. This is a
thinking-partner step: **write no files and create no branches.**

Idea / topic: $ARGUMENTS

1. Read the relevant code and the current `openspec/specs/` to understand how things work
   today. If `openspec/` does not exist yet, say so — the first `/sswa:propose` will
   scaffold it.
2. Lay out the options/approaches with trade-offs. Identify risks, unknowns, and anything
   that makes this *more than one feature*. SSWA ships one feature at a time — if the idea
   is several features, help split it and recommend which to do first.
3. Ask any clarifying questions needed to write a crisp proposal.
4. End by recommending a single, well-scoped change and a kebab-case `<change-name>`, and
   suggest: `/sswa:propose <change-name>` when ready.

Do not modify the repo in this mode.
